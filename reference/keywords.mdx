This page lists all reserved keywords in Ivy and their usage. Keywords are reserved identifiers that have special meaning in the language and cannot be used as variable, function, or type names.

## Keyword List

Ivy has **23** reserved keywords:

- `and`
- `as`
- `do`
- `else`
- `end`
- `false`
- `fn`
- `for`
- `from`
- `if`
- `impl`
- `import`
- `let`
- `match`
- `module`
- `mut`
- `or`
- `pub`
- `then`
- `trait`
- `true`
- `type`
- `where`
- `with`

## Declaration Keywords

### `let`

Introduces a variable binding.

```ivy
let x = 42;
let name = "Alice";
let (a, b) = (1, 2);
```

**Usage:**
- Top-level bindings
- Expression-level bindings
- Pattern matching bindings

### `mut`

Marks a variable as mutable.

```ivy
let mut counter = 0;
counter = counter + 1;
```

**Usage:**
- Must be used with `let`
- Allows reassignment via `=` operator

### `fn`

Defines a function.

```ivy
fn add(x, y) => x + y;

fn factorial(n)
  | n == 0 => 1
  | n > 0 => n * factorial(n - 1);
```

**Usage:**
- Top-level function declarations
- Lambda expressions: `fn (x) => x * 2`
- Trait and impl methods

### `type`

Defines a new type.

```ivy
type Option<a> = | None | Some(a);

type Person = {
  name: String,
  age: Int
};
```

**Usage:**
- Sum types (algebraic data types)
- Record types
- Type aliases

### `trait`

Defines a trait (interface).

```ivy
trait Show<a> {
  fn show: a -> String;
}

trait Eq<a> {
  fn eq(x: a, y: a): Bool;
}
```

**Usage:**
- Define abstract interfaces
- Specify type constraints

### `impl`

Implements a trait for a type.

```ivy
impl Show for Int {
  fn show(x) => int_to_string(x);
}

impl Eq for Point where Eq<Float> {
  fn eq(p1, p2) => p1.x == p2.x && p1.y == p2.y;
}
```

**Usage:**
- Implement trait methods
- Add behavior to types

### `module`

Declares the current module.

```ivy
module Math.Geometry;
```

**Usage:**
- Must appear at the top of a file
- Defines the module namespace

### `import`

Imports items from other modules.

```ivy
import Math;
import Math as M;
```

**Usage:**
- Import entire modules
- Import with aliases

### `from`

Imports specific items from a module.

```ivy
from Math import sin, cos, pi;
from List import *;
```

**Usage:**
- Selective imports
- Wildcard imports with `*`

### `as`

Creates an alias for an imported module.

```ivy
import VeryLongModuleName as Short;
```

**Usage:**
- Rename imports
- Avoid naming conflicts

### `pub`

Makes a declaration public.

```ivy
pub type Point = { x: Float, y: Float };

pub fn distance(p1, p2) => 
  sqrt((p2.x - p1.x)^2 + (p2.y - p1.y)^2);

pub let PI = 3.14159;
```

**Usage:**
- Export functions, types, and values
- Control module visibility

### `where`

Specifies type constraints.

```ivy
impl Show for List<a> where Show<a> {
  fn show(list) => show_list(list);
}
```

**Usage:**
- Add trait bounds in implementations
- Constrain generic types

## Control Flow Keywords

### `if`, `then`, `else`

Conditional expressions.

```ivy
if x > 0 then "positive" else "non-positive"

if condition then
  doSomething()
else
  doSomethingElse()
```

**Usage:**
- All three keywords must be used together
- `if` expressions always return a value
- Both `then` and `else` branches are required

<Note>
Unlike many languages, Ivy's `if` is an expression, not a statement. It always evaluates to a value.
</Note>

### `match`, `with`, `end`

Pattern matching expressions.

```ivy
match option with
  | None -> "nothing"
  | Some(x) -> "got " ++ show(x)
end

match list with
  | [] -> 0
  | x :: xs -> x + sum(xs)
end
```

**Usage:**
- All three keywords work together
- `match` starts the expression
- `with` introduces the pattern arms
- `end` terminates the match block

### `do`

Sequences multiple expressions.

```ivy
do {
  println("Starting");
  let x = compute();
  println("Done");
  x
}
```

**Usage:**
- Execute multiple expressions in sequence
- Last expression is the return value
- Semicolons separate expressions

### `for`

*Reserved for future use.* Currently not implemented.

```ivy
// Planned syntax (not yet implemented)
for x in list do
  println(x)
end
```

## Logical Keywords

### `and`

Logical AND operator (keyword form).

```ivy
if x > 0 and x < 10 then
  println("in range")
else
  println("out of range")
```

**Equivalent to:** `&&`

**Type:** `Bool -> Bool -> Bool`

**Short-circuiting:** Yes, the right operand is not evaluated if the left is `false`.

### `or`

Logical OR operator (keyword form).

```ivy
if x == 0 or x == 1 then
  println("zero or one")
else
  println("other")
```

**Equivalent to:** `||`

**Type:** `Bool -> Bool -> Bool`

**Short-circuiting:** Yes, the right operand is not evaluated if the left is `true`.

## Literal Keywords

### `true`

Boolean true literal.

```ivy
let flag = true;
let result = true && false;  // false
```

**Type:** `Bool`

### `false`

Boolean false literal.

```ivy
let flag = false;
let result = false || true;  // true
```

**Type:** `Bool`

## Comparison Table

### Operator vs. Keyword Forms

Some logical operations have both symbolic and keyword forms:

| Operation | Symbolic | Keyword | Example |
|-----------|----------|---------|----------|
| Logical AND | `&&` | `and` | `a && b` or `a and b` |
| Logical OR | `||` | `or` | `a || b` or `a or b` |

Both forms are equivalent and can be used interchangeably based on preference.

## Usage Notes

### Keywords Cannot Be Used as Identifiers

```ivy
// Invalid: keywords cannot be variable names
let let = 10;        // ERROR
let type = "hello";  // ERROR
let if = true;       // ERROR

// Valid: these are not keywords
let letter = 10;
let typed = "hello";
let condition = true;
```

### Case Sensitivity

All keywords are lowercase. Uppercase versions are not keywords:

```ivy
let Let = 10;        // OK (not a keyword)
let TYPE = "hello";  // OK (not a keyword)

type Option<a> = | None | Some(a);  // OK
Type Option<a> = | None | Some(a);  // ERROR (Type is not valid)
```

### Context-Dependent Usage

Some keywords only have meaning in specific contexts:

<Accordion title="Keyword Contexts">

**Top-level only:**
- `module` - only at file start
- `import`, `from`, `as` - only for imports
- `pub` - only for top-level declarations

**Expression context:**
- `if`, `then`, `else` - conditional expressions
- `match`, `with`, `end` - pattern matching
- `let` - bindings (both top-level and expression)
- `do` - block expressions

**Type/trait context:**
- `type` - type definitions
- `trait` - trait definitions
- `impl` - trait implementations
- `where` - type constraints

**Modifier keywords:**
- `mut` - mutability modifier
- `pub` - visibility modifier

</Accordion>

## Reserved but Unimplemented

### `for`

The `for` keyword is reserved for future iteration constructs but not currently implemented.

```ivy
// Planned (not yet available)
for item in collection do
  process(item)
end
```

**Current alternative:** Use recursion or higher-order functions like `map`, `filter`, `fold`.

## Examples

### Complete Program Using Multiple Keywords

```ivy
module Main;

import Math;
from List import map, filter;

pub type Result<t, e> = 
  | Ok(t) 
  | Err(e);

pub fn divide(a: Float, b: Float): Result<Float, String> =>
  if b == 0.0 then
    Err("Division by zero")
  else
    Ok(a / b);

fn process(numbers) => do {
  let filtered = filter(fn (x) => x > 0, numbers);
  let doubled = map(fn (x) => x * 2, filtered);
  doubled
};

let mut count = 0;

pub fn main() => match divide(10.0, 2.0) with
  | Ok(result) -> do {
      println("Result: " ++ show(result));
      count = count + 1;
      result
    }
  | Err(msg) -> do {
      println("Error: " ++ msg);
      0.0
    }
end;
```

This example demonstrates:
- Module and import keywords
- Type definition with `type`
- Public declarations with `pub`
- Functions with `fn`
- Conditionals with `if`/`then`/`else`
- Pattern matching with `match`/`with`/`end`
- Block expressions with `do`
- Mutable variables with `let mut`
- Lambda expressions
